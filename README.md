# 預金差押予定一覧 作成ツール（deposit_seizure_list）GUI版 機能要件書

## 1. ツールの目的

pipitLINQ経由で取得した電子預金照会結果（.xlsm）から、差押対象口座の情報を職員が対話的に選択し、WinActorで読み込み可能なCSV（預金差押予定一覧.csv）を生成する。同時に、差押起案の印刷・証跡用として照会結果ファイルを文書番号付きで保存する。現在職員が手作業で行っている口座選択・差押文言作成・一覧転記・ファイル整理を半自動化し、ミスの削減と処理速度の向上を実現する。

本書はWPF GUI版（.cs/.exe）の機能要件を記述する。ビジネスロジック（Excel読取・差押文言生成・CSV出力・印刷ファイル保存等）はコンソール版（.ps1）と同等であり、コンソール版の要件書も併せて参照のこと。

## 3. 起動方式

### 3.1 直接起動
- deposit_seizure_list.exe をダブルクリックで起動する。
- 初期画面（ドラッグ＆ドロップゾーン）が表示される。

### 3.2 ドラッグ＆ドロップ起動
- .xlsm ファイルを exe またはショートカットにD&Dして起動する。
- 複数ファイルを一度に処理可能（順次処理）。

### 3.3 コマンドライン引数起動
- file_search.exe の「差押リストに追加」から呼び出される。
- 引数で渡されたファイルパスを処理キューに追加する。
- `isFromFileSearch = true` となり、全ファイル処理完了後にアプリを自動クローズする。

## 4. 画面構成

### 4.1 ウィンドウ
- タイトル：「預金差押予定一覧 作成ツール」
- サイズ：960×680（最小 800×600）
- 3つの画面状態（InitialPanel / MainPanel / OverlayPanel）を重ね合わせた Grid で構成する。

### 4.2 初期画面（InitialPanel）
- 中央にドラッグ＆ドロップゾーン（破線枠 + フォルダアイコン）を配置する。
- 「ファイルを選択」ボタンでファイル選択ダイアログも使用可能。
- ウィンドウ全体へのD&Dに対応し、.xlsm ファイルのみ受け付ける。

### 4.3 メインフォーム（MainPanel）
- ヘッダーバー（ティール、ツール名表示）
- シート選択 ComboBox + ファイルリンク + 再読み込みボタン
- 基本情報セクション（宛名番号・氏名・金融機関・処分担当・住民票住所・届出住所・執行日）
- 口座選択セクション（ListView + GridView）
- フッターバー（件数表示 + 出力先パス）
- アクションボタン群（ファイルを読み込む / スキップ / 一覧に追加）

### 4.4 オーバーレイ（OverlayPanel）
- 半透明白背景（#CCFFFFFF）でメインフォーム全体を覆う。
- ローディングオーバーレイ：回転スピナー（青弧の Path + RotateTransform）
- 結果オーバーレイ：処理結果カード

## 5. 基本情報セクション

### 5.1 読取専用フィールド
- 宛名番号（10桁0埋め、Consolas フォント、グレー背景）
- 金融機関名

### 5.2 編集可能フィールド
- 氏名、処分担当、住民票住所（✎マーク付きラベル）
- 届出住所（チェックボックス連動、50文字制限バリデーション付き）

### 5.3 執行日入力
- テキスト入力（Consolas フォント）+ カレンダーPopupボタン。
- 7桁和暦（例: 5080615）または西暦（例: 2026/06/15）を受け付ける。
- LostFocus 時に自動パース・バリデーションを実行し、不正値は赤枠で警告する。

### 5.4 カレンダーPopup
- Popup + Calendar コントロール（SelectionMode: SingleDate）。
- 日付選択時に自動で Popup を閉じ、入力欄に反映する。
- Popup 展開時に DisplayMode を Month にリセットし、前回の年/月選択ビューが残らないようにする。
- SelectedDates をクリアしてから開くことで、同じ日付の再クリックでも SelectedDatesChanged が発火する。

## 6. 口座選択セクション

### 6.1 口座テーブル（ListView + GridView）
- 6列：支店名 / 支店番号 / 口座種別 / 最終取引日（満期日）/ 口座番号 / 残高
- 口座番号に差押実績がある場合は「★」マークを付与し、ToolTip に文書番号を表示する。
- 選択行はティール背景（#E0F2F1）＋左端に3pxアクセントバーで強調する。
- ウィンドウリサイズ時に列幅を自動調整する（SizeChanged イベント）。

### 6.2 ガイドテキスト
- 口座が0件の場合：「表示可能な口座がありません」
- 口座未選択の場合：「口座を選択してください」
- 執行日未入力の場合：「執行日を入力してください」
- すべて満たされた場合：ガイドテキスト非表示、「一覧に追加」ボタン有効化

## 7. 非同期処理

### 7.1 BackgroundWorker パターン
- Excel COM 操作はすべて BackgroundWorker で非同期実行する。
- UI スレッドのブロックを防ぎ、処理中はローディングオーバーレイ（スピナー）を表示する。

### 7.2 非同期処理の発生箇所
| 処理 | トリガー | BackgroundWorker の内容 |
|------|---------|----------------------|
| ファイル読込 | D&D / ファイル選択 / 次のファイルへ | ReadExcelFile（COM でファイルオープン→シートデータ読取） |
| シート切替 | ComboBox 選択変更 | ReadSheetData（選択シートのデータ再読取） |
| 一覧に追加 | ボタン押下 | ProcessAdd（文書番号採番→差押文言生成→CSV追記→印刷ファイル保存） |

### 7.3 冗長読込の抑止
- PopulateForm 内で sheetCombo.SelectedIndex を設定する際、suppressSheetChange フラグで SelectionChanged を抑止する。
- ReadExcelFile で初回シートのデータを既に取得済みのため、二重の COM アクセスを回避する。

## 8. アニメーション

### 8.1 スピナー回転
- ローディングオーバーレイ中央に270°の青弧（Path + ArcSegment）を表示する。
- RotateTransform + DoubleAnimation（By=360, 1秒, Forever）で無限回転する。
- loadingOverlay の IsVisibleChanged イベントで自動的に開始/停止し、非表示時のリソース消費を防止する。

### 8.2 オーバーレイフェード
| メソッド | 動作 | 時間 |
|----------|------|------|
| FadeInOverlay | overlayPanel の Opacity 0→1 | 150ms |
| FadeOutOverlay | overlayPanel の Opacity 1→0 → Collapsed | 150ms |
| CrossFadeToResult | overlayPanel を一瞬フェードアウト → コンテンツ差替 → フェードイン | 100ms × 2 |

### 8.3 フェードの使い分け
| 場面 | 方式 |
|------|------|
| ローディング開始 | FadeInOverlay |
| ファイル読込成功 / シート切替完了 | FadeOutOverlay |
| ローディング → 結果カード | CrossFadeToResult |
| スキップ（メインフォームから直接結果表示） | ShowResult + FadeInOverlay |
| 次のファイルへ / 完了 | FadeOutOverlay（コールバック付き） |

## 13. データ加工処理

コンソール版と同等のビジネスロジックを BusinessLogic クラスに集約している。

### 13.1 ゆうちょ銀行対応
- yucho_center_mapping.json により、支店番号（記号）の2〜3桁目から貯金事務センター名を自動補完する。
- 口座カテゴリ（総合口座 / 振替口座）により参照テーブルを切り替える。

### 13.2 支店名「支店」サフィックス
- ゆうちょ以外の金融機関で、支店名の末尾に「支店」がない場合に自動付与する。
- 末尾が「営業部」「出張所」「公務部」「本店」の場合は付与しない。

### 13.3 金融機関名・口座種別マッピング
- institution_name_mapping.json により金融機関名を補正する。
- account_type_mapping.json により口座種別を補完する（global + 金融機関別の2階層）。
- branch_name_mapping.json により支店名を補正する。
- いずれも任意ファイルで、存在しない場合は補正なし。

### 13.4 全角変換
- 金融機関名のマッピング比較時に半角→全角正規化（ToFullWidth）を行い、表記ゆれを吸収する。

## 14. 文書番号の採番

### 14.1 排他ロック
- document_number_counter.json を FileStream 排他ロック（FileShare.None）で開き、「読み取り→採番→書き戻し」をアトミックに実行する。
- 複数職員が同時使用した場合の重複採番を防止する。

### 14.2 リトライ
- ロック取得に失敗した場合は最大5回、500ms間隔でリトライする。

### 14.3 CSV書き込み失敗時のロールバック
- CSV書き込みが全リトライ失敗した場合、消費済みの文書番号カウンターを1つ戻す。

---

## 15. 差押文言の生成

### 15.1 差押文言の構成
- 3つのフィールド（差押文言1〜3）に分割して出力する。
- 文言は「上記滞納者が、債務者である〜」形式で構成する。
- 金融機関名には「扱」を付与する。

### 15.2 ゆうちょ銀行の特殊形式
- ゆうちょ銀行の場合は「記号番号：支店番号－口座番号」形式。
- その他は「口座番号：口座番号」形式。

### 15.3 振込手数料
- transfer_fee_mapping.json で金融機関ごとの振込手数料文言を差押文言2に出力する。

## 17. 照会結果ファイルの保存

### 17.1 保存先
- printFolder（config で設定）配下に「文書番号.xlsm」として保存する。

### 17.2 不要データの削除
- 選択シート以外のシートを削除する（VeryHidden シートは保持）。
- 選択した口座に対応しない表紙ブロック（カバーページ＋口座テーブル）を削除する。
- 明細ページの削除：detailAccountNumberCell / detailAccountTypeCell で口座を照合し、選択口座以外の明細ページを削除する。

## 20. コンソール版との主な差異

| 項目 | コンソール版（.ps1） | GUI版（.cs/.exe） |
|------|-------------------|------------------|
| 実行環境 | PowerShell | .NET Framework 4.0 WPF |
| 起動方法 | bat → PowerShell | bat → csc.exe → exe |
| ユーザー操作 | テキスト入力（番号選択・Y/N/R） | GUI操作（クリック・D&D・カレンダー） |
| シート選択 | 番号入力 | ComboBox |
| 口座選択 | 番号入力 | ListView クリック |
| 執行日入力 | テキスト入力（元号/西暦） | テキスト入力 + カレンダーPopup |
| 確認画面 | Y/N/R の3択 | 「一覧に追加」/「スキップ」ボタン |
| 処理中表示 | なし（同期実行） | スピナー + フェードアニメーション |
| ログ | トランスクリプトファイル出力 | なし（結果カードで即時フィードバック） |
| サマリー | 処理完了時に一覧表示 | なし（1件ずつ結果カードで表示） |
| file_search 連携 | PowerShell 経由 + 環境変数 | exe 直接起動（引数にパスを渡す） |
